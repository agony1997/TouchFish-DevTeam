  ├── repository/      ← UserRepository, TaskRepository

  ├── controller/      ← AuthController, TaskController
  ├── security/        ← JwtTokenProvider, JwtAuthenticationFilter
  └── exception/       ← GlobalExceptionHandler, custom exceptions

  Frontend (Vue 3 + TypeScript + Pinia)
  src/
  ├── api/             ← apiClient.ts (axios + interceptor), taskApi.ts
  ├── stores/          ← authStore.ts, taskStore.ts
  ├── components/      ← TaskFilters, TaskListItem, TaskCreateForm, Pagination
  ├── views/           ← TaskListPage.vue
  └── router/          ← 新增 /tasks 路由 + auth guard

  團隊分工與依賴流

  ┌──────────────┬────────────────────────┬─────────────────────────────────┐
  │    Agent     │        任務順序        │            預期產出             │
  ├──────────────┼────────────────────────┼─────────────────────────────────┤
  │              │ M1(Auth) → M2(CRUD) →  │ 7 entities/DTOs, 2 controllers, │
  │ backend-dev  │ M3(State Machine)      │  2 services, JWT filter,        │
  │              │                        │ exception handler               │
  ├──────────────┼────────────────────────┼─────────────────────────────────┤
  │              │ API client → authStore │ axios interceptor, 2 stores, 4  │
  │ frontend-dev │  → taskStore → 元件群  │ components, 1 view, router 更新 │
  │              │ → 頁面整合             │                                 │
  └──────────────┴────────────────────────┴─────────────────────────────────┘

  Frontend 不需等 Backend 完成再開始 — 可先根據 spec 定義好 API client 介面和
  Store，最後整合。

  關鍵設計決策

  1. 軟刪除: Task entity 使用 deleted boolean 欄位 + @Where(clause = "deleted = 
  false")

  3. 分頁: 使用 Spring Data 的 Pageable，轉換為自定義分頁 DTO 回傳
  4. 前端篩選: 所有篩選走後端 API（server-side filtering），非前端過濾
