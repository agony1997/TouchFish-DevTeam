## 1. Executive Summary

Delivered a full-stack task management application with authentication, task CRUD, state machine, and a task list UI. The backend is built on Spring Boot 3.2.3 with H2 in-memory database, JWT-based security, and soft-delete support. The frontend uses Vue 3 with Pinia state management, Axios HTTP client, and Vue Router. All 8 tasks completed successfully with 272/272 tests passing (173 backend + 99 frontend) across 4 modules and 63 acceptance criteria.

## 3. API Contract (Final)

### Endpoints

| Method | Path | Request | Response | Errors |
|--------|------|---------|----------|--------|
| POST | /api/auth/register | `{username: string(3-50, alphanumeric+underscore), password: string(min 8), displayName: string(1-100)}` | `{id: long, username: string, displayName: string}` (201) | 400, 409 |
| POST | /api/auth/login | `{username: string, password: string}` | `{token: string, expiresIn: number(86400)}` (200) | 401 |
| GET | /api/auth/me | Authorization: Bearer token | `{id: long, username: string, displayName: string}` (200) | 401 |
| POST | /api/tasks | `{title: string(1-200, required), description: string(max 5000, optional), priority: enum(HIGH\|MEDIUM\|LOW, optional, default MEDIUM)}` | TaskResponse (201) | 400, 401 |
| GET | /api/tasks/{id} | — | TaskResponse (200) | 401, 404 |
| PUT | /api/tasks/{id} | `{title: string(1-200, required), description: string(max 5000, optional), priority: enum(HIGH\|MEDIUM\|LOW, optional)}` | TaskResponse (200) | 400, 401, 404 |
| DELETE | /api/tasks/{id} | — | 204 No Content | 401, 404 |
| GET | /api/tasks | Query: `page(0), size(20), sort(createdAt,desc), keyword, priority` | PageResponse\<TaskResponse\> (200) | 401 |
| PATCH | /api/tasks/{id}/status | `{status: enum(TODO\|IN_PROGRESS\|DONE\|CLOSED)}` | TaskResponse (200) | 400, 401, 404 |
| GET | /api/tasks/{id}/transitions | — | `{currentStatus: enum, availableTransitions: enum[]}` (200) | 401, 404 |

## 7. Known Issues & Future Work

- [ ] **OBS-1: SecurityConfig 401 response format** (LOW) — The `authenticationEntryPoint` returns `{"message":"Unauthorized"}` without a `timestamp` field, inconsistent with the standard `ErrorResponse {message, timestamp}` format. This is a cosmetic issue since Spring Security generates this before reaching the controller layer.
- [ ] **OBS-2: Router uses PlaceholderComponent for /tasks** (EXPECTED) — The `/tasks` route maps to a PlaceholderComponent instead of TaskListPage.vue. TaskListPage is tested via direct mounting and will be wired when the full application is assembled.
- [ ] **OBS-3: Combined keyword+priority filtering is exclusive** (LOW) — TaskService uses if/else-if logic, so keyword takes precedence over priority when both are provided. No AC explicitly requires combined filtering, but the frontend UI allows setting both simultaneously.
- [ ] **OBS-4: Frontend status filter sent but backend ignores it** (LOW) — The taskStore sends a `status` query parameter, but TaskController.listTasks() does not accept it. The parameter is silently ignored. TaskRepository has `findByDeletedFalseAndStatus()` but it is not wired in the service.
- [ ] **OBS-5: Priority not enum-validated at API level** (LOW) — TaskRequest.priority is a plain String without enum validation. Invalid values like "INVALID" would be accepted and stored. No AC explicitly requires rejecting invalid priority values.
