## 3. API Contract (Final)

### Endpoints

| Method | Path | Request | Response | Errors |
|--------|------|---------|----------|--------|
| POST | /api/auth/register | `{username: string(3-50, ^[a-zA-Z0-9_]+$), password: string(min 8), displayName: string(1-100)}` | 201: `{id: long, username: string, displayName: string}` | 400 (validation), 409 (duplicate username) |
| POST | /api/auth/login | `{username: string, password: string}` | 200: `{token: string, expiresIn: 86400}` | 401 (bad credentials) |
| GET | /api/auth/me | Bearer token | 200: `{id: long, username: string, displayName: string}` | 401 (unauthorized) |
| POST | /api/tasks | `{title: string(1-200, required), description?: string(max 5000), priority?: enum(HIGH\|MEDIUM\|LOW, default MEDIUM)}` Bearer | 201: TaskResponse | 400 (validation), 401 |
| GET | /api/tasks/{id} | Bearer token | 200: TaskResponse | 401, 404 |
| PUT | /api/tasks/{id} | `{title: string(1-200, required), description?: string(max 5000), priority?: enum}` Bearer | 200: TaskResponse | 400, 401, 404 |
| DELETE | /api/tasks/{id} | Bearer token | 204: (no body) | 401, 404 |
| GET | /api/tasks | `?page=0&size=20&sort=createdAt,desc&keyword=&priority=&status=` Bearer | 200: TaskPageResponse | 401 |
| PATCH | /api/tasks/{id}/status | `{status: enum(TODO\|IN_PROGRESS\|DONE\|CLOSED)}` Bearer | 200: TaskResponse | 400 (invalid transition), 401, 404 |
| GET | /api/tasks/{id}/transitions | Bearer token | 200: TransitionResponse | 401, 404 |

### Shared Types

| Type | Fields |
|------|--------|
| TaskResponse | `id: long, title: string, description: string?, priority: enum(HIGH\|MEDIUM\|LOW), status: enum(TODO\|IN_PROGRESS\|DONE\|CLOSED), createdBy: long, createdAt: ISO8601, updatedAt: ISO8601?` |
| TaskPageResponse | `content: TaskResponse[], page: int, size: int, totalElements: long, totalPages: int` |
| TransitionResponse | `currentStatus: string, availableTransitions: string[]` |

### State Machine Rules

| From | Allowed Transitions |
|------|-------------------|
| TODO | IN_PROGRESS |
| IN_PROGRESS | TODO, DONE |
| DONE | CLOSED |
| CLOSED | (terminal -- no transitions) |

### Error Format

All error responses: `{"message": "..."}`  with appropriate HTTP status code.

## 7. Known Issues & Future Work

1. **Duplicate TaskResponse mapping** -- `TaskController.changeStatus()` constructs TaskResponse inline rather than reusing `TaskService.toResponse()` (which is private). Fields are identical; this is a minor refactor opportunity with no functional impact.

2. **No task detail page route** -- Frontend router defines `/tasks` (list) but no `/tasks/:id` (detail). `TaskListPage.onTaskClick()` navigates to `/tasks/${task.id}` which won't match a route. This is expected -- task detail page is outside M6 scope and would be a future milestone.

3. **LoginResponse.expiresIn hardcoded** -- `AuthService.login()` returns `new LoginResponse(token, 86400)` with a hardcoded value rather than deriving from `JwtTokenProvider`'s configured expiration. The value matches the CONTRACT (86400 seconds = 24 hours) so there is no functional issue, but a future change to token expiry duration would need to update both places.
